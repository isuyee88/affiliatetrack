### 1.2 技术栈选型

| 组件 | 技术选型 | 选型理由 |
|------|----------|----------|
| **边缘计算** | Cloudflare Workers | 全球310+数据中心，<50ms延迟，免费10万次请求/天 |
| **数据库** | Cloudflare D1 | SQLite兼容，边缘部署，免费5GB存储，每日10万次读取 |
| **状态管理** | Durable Objects | 有状态边缘计算，支持WebSocket，免费100万次请求/月 |
| **对象存储** | Cloudflare R2 | S3兼容，免费10GB存储，每月100万次操作 |
| **缓存** | Cloudflare Cache API | 边缘缓存，减少数据库查询 |
| **前端框架** | React + TypeScript | 类型安全，组件化，生态丰富 |
| **构建工具** | Vite | 快速构建，优化输出 |
| **API框架** | Hono.js | 轻量级，Workers优化，TypeScript原生 |
| **ORM** | Drizzle ORM | 类型安全，D1支持，轻量级 |
| **样式** | Tailwind CSS | 原子化CSS，快速开发 |
| **图表** | Recharts | React原生，灵活定制 |

## 5. 架构决策记录 (ADR)

### ADR-001: 选择Cloudflare Workers而非传统服务器
**决策**: 使用Cloudflare Workers作为计算平台  
**原因**:
- 全球边缘部署，延迟极低
- 自动扩展，无需运维
- 免费额度充足
- 与D1、R2原生集成

### ADR-002: 选择D1作为数据库
**决策**: 使用Cloudflare D1作为关系数据库  
**原因**:
- 边缘部署，延迟低
- SQLite兼容，开发简单
- 免费额度满足初期需求
- 与Workers原生集成

### ADR-003: 使用Durable Objects管理状态
**决策**: 使用Durable Objects管理会话和实时状态  
**原因**:
- 有状态边缘计算
- 支持WebSocket
- 避免Workers冷启动
- 实时聚合能力
